# epso 常见问题（FAQ）

> 这里整理了使用 epso 过程中最常见的问题和对应解决思路。

---

## 1. 触发词输入后没有替换？

- 检查 **无障碍服务** 是否已开启。
- 检查首页或设置中 **文本扩展** 开关是否打开。
- 检查规则是否已启用，以及是否被 `filter_exec` / `filter_title` 过滤。
- 检查应用是否被电池优化限制，建议加入白名单。

## 2. 包商店加载失败或下载很慢？

- 国内访问 GitHub 可能受限，尝试开启 **包商店走代理镜像** 开关。
- 也可以在浏览器手动下载包 zip 后，通过包商店顶栏的 **文件夹图标** 从本地导入。
- 高级用户还可以在 **设置 → 包商店索引源** 填入自定义镜像地址。

## 3. 安装包后规则没生效？

- 安装后 epso 会自动在工作区 `matches/packages/{name}/package.yml` 中创建规则。
- 返回首页或规则列表，确认包已出现在 `imports` 中。
- 如果仍不生效，尝试 **重新加载工作区** 或重启无障碍服务。

## 4. 重启手机后服务不运行？

- Android 重启后无障碍服务需要用户手动重新开启，系统不允许应用自动打开。
- epso 会在开机后尝试恢复 **状态服务** 的通知，并提醒你检查无障碍开关。
- 将 epso 加入 **电池优化白名单** 可降低服务被系统杀死的概率。

## 5. 脚本/JS 变量输出为空？

- `script` 类型依赖设备已安装的解释器（如 Termux 的 Python/Node）。
- `javascript` 类型使用内置 Rhino 引擎，确保 `code` 字段最后一行是返回值。
- 可在规则编辑页点击变量卡片查看日志或开启 `debug: true`。

## 6. 同步冲突了怎么办？

- 在 **同步设置** 中选择冲突策略：
  - `LastWriteWins`：以最后修改的版本为准。
  - `KeepBoth`：保留两个版本，冲突文件会生成 `.conflict` 后缀。
- 建议开启 **保存后自动推送**，减少冲突概率。

## 7. 如何关闭所有替换？

- 在首页或设置中关闭 **文本扩展（Text expansion）** 开关。
- 或者临时在通知栏点击磁贴 / 快速设置磁贴关闭。
