# epso 故障排查指南

> 当 epso 表现异常时，按以下步骤定位问题。

---

## 一、服务状态检查

1. 打开 epso 首页，确认顶部状态为 **已就绪 / 服务正在运行**。
2. 如果显示 **无障碍未授权**，进入系统设置开启 epso 的无障碍服务。
3. 如果显示 **无障碍已关闭**，点击首页开关或通知栏磁贴重新启用。
4. 如果显示 **权限受限**，说明被 `appOps` 限制，需通过 Shizuku 或 adb 恢复。

## 二、规则不生效

1. 进入 **规则** 页面，确认规则存在且已启用。
2. 检查规则是否有 `filter_exec` / `filter_title` / `filter_class` 过滤。
3. 检查触发词是否包含默认前缀（默认 `:`），或规则是否覆盖了 `prefix`。
4. 查看规则编辑页的 **测试** 按钮，确认能正常展开。
5. 检查 YAML 文件是否被正确加载（文件页 → 重新加载）。

## 三、包商店/网络问题

1. 在设置中开启 **包商店走代理镜像**。
2. 尝试刷新包列表，观察错误提示：
   - **连接超时**：网络无法访问 GitHub，建议切换网络或开启代理。
   - **DNS/证书错误**：可能是 DNS 污染或系统时间不正确。
   - **404**：上游索引正在更新，可稍后再试。
3. 作为备用方案，从浏览器下载 `.zip` 包后，通过包商店顶栏 **导入** 按钮安装。

## 四、同步异常

1. 在 **同步设置** 中点击 **测试连接**，确认 WebDAV/本地文件夹可访问。
2. 检查冲突策略是否符合预期，冲突文件会以 `.conflict` 后缀保留。
3. 如果自动同步不触发，确认已开启 **保存后自动推送** 和通知权限。
4. WebDAV 服务器需支持 `PROPFIND` / `PUT` / `GET` / `DELETE` 方法。

## 五、耗电/被杀后台

1. 进入 **设置 → 电池优化白名单**，将 epso 设为不优化。
2. 在最近任务中锁定 epso（部分系统需要）。
3. 关闭系统对 epso 的省电策略（如 MIUI 省电限制、华为应用启动管理）。

## 六、日志与反馈

- 开启 `debug: true` 可在变量求值时输出详细日志。
- HTTP API `/api/v1/expand` 可用于无头调试触发和替换结果。
- 如问题持续，请导出工作区 ZIP 并在 GitHub Issues 提交。
